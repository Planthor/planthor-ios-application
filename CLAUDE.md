## Commands

```bash
# Install dependencies
flutter pub get

# Run code generation (required after any @riverpod annotation or model change)
dart run build_runner build --delete-conflicting-outputs

# Analyze (must pass zero errors before PR)
flutter analyze

# Run app
flutter run --dart-define=ENV=dev        # dev (local Docker Compose)
flutter run --dart-define=ENV=prod       # prod (cloud)

# Run all tests
flutter test

# Run single test file
flutter test test/widget_test.dart
```

## Architecture

Feature-first Clean Architecture with Riverpod state management.

**Entry point:** `lib/main.dart` wraps `MyApp` in `ProviderScope`, watches `authProvider`, routes to `SignInScreen` or `MainScaffold`.

**Navigation flow:** `SignInScreen` → `MainScaffold` (`PlanthorBottomNav` — 4 tabs: Home, Plans, Community, Settings). `MainScaffold` watches `navigationProvider` (generated Riverpod notifier) to switch between `DiscoveryScreen`, `PlansScreen`, `CommunityScreen` (stub), and `ProfileScreen`. Profile pushes to `PersonalInformationScreen` and `ConnectAppsScreen`.

**Feature structure** — each feature under `lib/features/<name>/`:
```
presentation/
  screens/      # Full-page widgets (ConsumerWidget)
  widgets/      # Feature-scoped UI components
  providers/    # @riverpod class notifiers (with codegen)
  bloc/         # Plain FutureProvider declarations (no codegen)
domain/
  entities/     # Plain Dart data classes, fromJson factories
  repositories/ # Abstract interfaces
data/
  datasources/  # HTTP, secure storage, platform APIs
  repositories/ # Concrete implementations
```

**Core layer** (`lib/core/`):
- `config/` — `AppConfig` (env-aware Keycloak + API URLs, toggled via `--dart-define=ENV=dev|prod`)
- `network/` — `apiClientProvider` (Dio + auth interceptor + log interceptor)
- `theme/` — `AppColors`, `AppTheme` (Riverpod-generated, Material 3, forest-green palette; includes plan status + Strava brand tokens)
- `widgets/` — `PlanthorAppBar` (64px, avatar, notification bell), `PlanthorBottomNav` (4-tab animated nav)
- `services/`, `utils/` — shared cross-feature code (Phase 3+)

**Code generation:** All `*.g.dart` files are generated by `build_runner`. Never edit them manually. Any file with `@riverpod` annotation or `part '*.g.dart'` directive requires a build_runner run after changes.

## Key Conventions

- Screens that read providers extend `ConsumerWidget` or `ConsumerStatefulWidget`
- Mutable state with actions → `@riverpod` class + codegen
- Read-only async data → plain `FutureProvider` (no codegen needed)
- `apiClientProvider` injects the Bearer token automatically — all feature providers just watch it
- PR titles use conventional commits: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`
- Branch naming: `feature/<name>` or `fix/<name>`
- Every new screen requires a widget test in `test/`
